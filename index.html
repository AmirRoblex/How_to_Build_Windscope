üíª Installing the Brains (Software Setup)

Let's get the software running on your Mini-PC. You'll need an internet connection and to be comfortable using the command line (Terminal on Mac/Linux, or PowerShell/CMD on Windows).

Step 1: Install the Prerequisites

Before we download the Windscope code, we need to install the tools it relies on.

  • Git: A tool for downloading code from GitHub. Download Git here.
  • Node.js: The runtime environment for the main simulator server. Download Node.js here (LTS version is recommended).
  • Python: The language used for the camera scanner. Download Python here. (Make sure to check "Add Python to PATH" during installation on Windows).

Step 2: Set Up the Scanner (Python)

This program uses the camera to identify the blocks on the grid. It's the "eyes" of your exhibit.

# 1. Open your terminal.

# 2. Clone (download) the scanner repository.

$ git clone https://github.com/Bachelor-thesis-windpark-simulator/scanner.git


# 3. Navigate into the new directory.

$ cd scanner


# 4. Install the necessary Python libraries.

$ pip install -r requirements.txt


# 5. VERY IMPORTANT: Calibrate the camera!

# The program needs to correct for the fisheye lens. Run the script and hold a chessboard pattern in front of the camera at different angles.

$ python calibrate.py

Step 3: Set Up the Simulator (Node.js)

This is the main application server. It's the "brain" that runs the simulation and creates the visuals.

# 1. Open a NEW, separate terminal window.

# 2. Clone the main simulator repository.

$ git clone https://github.com/Bachelor-thesis-windpark-simulator/windpark-simulator.git


# 3. Navigate into the new directory.

$ cd windpark-simulator


# 4. Install the Node.js dependencies. This might take a few minutes.

$ npm install

üöÄ Let's Go! Running the Project

You've built the hardware and installed the software. It's time to bring your Windscope to life! You'll need to have both terminal windows open from the previous step.

Step 1: Start the Scanner

In your **first** terminal (the one in the `scanner` directory), run the main Python script. This activates the camera.

$ python main.py

A window should pop up showing the live feed from your camera. You should be able to see the bottoms of the blocks you place on the grid.

Step 2: Start the Simulator Server

In your **second** terminal (the one in the `windpark-simulator` directory), start the Node.js server. This starts the simulation.

$ npm start

You'll see some log messages appear, indicating the server is running and waiting for the scanner.

Step 3: Open the Views in a Browser

The simulation is now running! Open a web browser (like Chrome or Firefox) on the Mini-PC.

  • Projector View: Navigate to http://localhost:8000/city. Drag this browser window onto the projector's display and make it **full screen** (usually by pressing F11).
  • Dashboard View: Open a new tab and navigate to http://localhost:8000/dashboard. Drag this window onto your dashboard monitor.

Congratulations!

Your Windscope is fully operational. Start placing blocks and see your wind park come to life!

üéÆ How to Play & Learn the Science

Experiment by placing blocks and turning the wind knobs. Watch how the dashboard changes in real-time. Here's the science behind what you're seeing:

The Power Curve: Wind Speed is Key!

Not all wind is the same. Wind turbines have a "sweet spot" for generating power. Use your wind speed knob and watch the Power (Leistung) on the dashboard.

  • Too Slow: Below the "cut-in speed" (Anlaufgeschwindigkeit, ~3 m/s), the blades won't even turn. No power!
  • Just Right: As speed increases, power skyrockets! It increases with the CUBE of the velocity.
  • Rated Speed: At a certain point (~14 m/s), the turbine reaches its maximum "rated power" (Nennleistung). Even if the wind gets faster, it won't produce more power, to protect itself.

The Wake Effect: Don't Block the Wind!

When a turbine captures wind, it leaves a "wind shadow" of slower, more turbulent air behind it. This is called the wake effect (Nachlauf). Placing another turbine in this wake is a bad idea!

  • Less Power: The downstream turbine gets less "fuel" (wind) and produces much less electricity.
  • Shorter Lifespan: The turbulent, choppy air causes extra wear and tear (Materialerm√ºdung), reducing the turbine's lifespan. Watch the Lifespan (Lebensdauer) smiley on your dashboard!

Try it! Place two turbines in a row and turn the wind direction knob so one is directly behind the other. See what happens to the Power and Lifespan values.

The wake of slower, turbulent air behind a turbine.

Distance Rules (Abstandsregelungen)

In the real world, there are strict rules about where you can build turbines to protect people and nature. The simulator includes these! If you place a turbine too close to something, a red ‚ùó will appear.

  • To Houses: Turbines must be a safe distance from settlements to protect residents from noise and "shadow flicker."
  • To Each Other: To avoid the wake effect, turbines need to be spaced out‚Äîtypically 5-7 times their rotor diameter apart in the main wind direction!
  • To Water/Roads: You also need to respect buffer zones around infrastructure and natural features.

The Jensen Wake Model: Simulating Wind Shadows

The "wake effect" isn't random; it's calculated using a simplified physics model developed by N.O. Jensen. The model assumes the wind shadow expands linearly behind the turbine. The simulation uses this formula to calculate the reduced wind speed ($v_1$) at a distance ($x$) behind a turbine:

üë©‚Äçüè´ Teacher's Hub: Using Windscope in the Classroom

Windscope is more than just a model; it's a dynamic sandbox for exploring complex topics in a tangible way. Here‚Äôs how you can leverage it for your classroom.

Learning Objectives

By the end of a session with Windscope, students should be able to:

  • Explain the relationship between wind speed and power generation (the power curve).
  • Define the "wake effect" and explain why it's a critical factor in wind park design.
  • Identify the social and environmental factors (like distance rules) that influence wind park planning.
  • Collaborate in a team to design a solution that balances competing goals (e.g., maximizing power vs. protecting turbine lifespan).

Teaching Physics & Science with Windscope

Use the exhibit to make abstract science concepts tangible and interactive.

  • Physics: Visually demonstrate the transformation of kinetic energy (wind) into electrical energy. The "Power" value on the dashboard is a direct measurement. Use the power curve to discuss concepts of work, power, and efficiency.
  • Geography & Environmental Science: The exhibit is a perfect tool for discussing land use conflicts. Why can't we build a wind park in a residential area or a nature reserve? Students must make decisions about site selection that balance energy needs with social and environmental protection.
  • Computer Science & Engineering: Discuss the system as an example of Human-Computer Interaction (HCI). How do the physical blocks (input) get translated into a digital simulation (output)? This is a great introduction to modeling, simulation, and the Internet of Things (IoT).

Guiding Questions to Spark Discussion

Use these questions to encourage critical thinking while students are using the exhibit:

  • "What happens to the total power when you cluster all the turbines close together vs. spreading them out? Why do you think that is?"
  • "If the wind is coming from the north, is it better to arrange your turbines in a horizontal line or a vertical line? Why?"
  • "Find the wind speed that gives you the most power. What happens if you go much faster than that? Is more wind always better?"
  • "Look at the 'Lifespan' smiley. What actions make it go down? How can you design a park that is both powerful and long-lasting?"
  • "Why do you think there are rules about how close turbines can be to houses? What problems might they cause in real life?"

ü§î Troubleshooting & FAQ

Running into a problem? Don't worry! Here are some solutions to common issues.

The projector image is blurry or not aligned.

1. Focus: Most projectors have a focus ring around the lens. Adjust it until the image is sharp.

2. Keystone Correction: If the image is trapezoidal, use the projector's keystone settings to make it a perfect rectangle.

3. Alignment: Physically move the projector until the projected grid perfectly overlaps with the physical grid on the plexiglass.

The scanner isn't detecting my blocks correctly.

1. Re-calibrate: The most common issue! Re-run the `python calibrate.py` script. Make sure to show the chessboard pattern at many different angles and distances.

2. Lighting: Ensure the area under the table is evenly lit and there are no strong reflections on the plexiglass.

3. Check Patterns: Double-check that the black and white patterns on the bottom of your blocks exactly match the guide.

The dashboard shows "0 kW" even with turbines on the board.

1. Check Wind Speed: Is the wind speed knob turned up? If the speed is below the "cut-in speed" (~3 m/s), the turbines won't produce power.

2. Server Connection: Make sure both the scanner script and the server script are running in their terminals without any errors.

One of the scripts crashes on startup.

1. Dependencies: Make sure you have installed all the required libraries by running `pip install -r requirements.txt` in the `scanner` directory and `npm install` in the `windpark-simulator` directory.

2. Port Numbers: Ensure no other application on your computer is using port 8000.

3. Ask for Help: If you're still stuck, post a question on the project's GitHub Issues page!
